- Update CONTRIBUTING.md when processes change
- Write CHANGELOG.md entries under [Unreleased] for each change

## Conventions

- Commit format for docs: `docs(scope): description`
- Documentation should be accurate, concise, and up-to-date
- Include command examples that actually work
- Grammar/typo fixes can be merged quickly; major content changes need discussion
- Never add AI attribution comments in source files — attribution belongs in git commits
